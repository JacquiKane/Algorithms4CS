Find Value In Sorted Array, with Binary Search

Array must be sorted! Let's use an example. Below, the array is called nums. We scan the array, looking for a value of 27.


Sample Code, Assuming Array Of Ints - nums


        int low = 0;
        int high = nums.length-1;
        int middle = (low+high)/2;
                        
        while (nums[middle] != target && low <= high) {
            if (target < nums[middle])
                high = middle - 1;
            else {
                low = middle + 1;
                middle = (low + high) / 2;
               }  
            if (nums[middle] == target)
                // Indicate found
            else
                // Indicate not found
            }

How Efficient Is Binary Search?

We say of the order of, designated by O(...), to describe the worst case in terms of comparisons. This is termed big O representation. The best case is described using big Omega notation, Ω(...).

The array has to be sorted before binary search can be used. This may take away from any efficiency gains! In the best case, the item is found right away, after a single array split. In this case, the best case scenario is described as big Omega of 1, Ω(1). In the worst case, we have to keep splitting the array, and end up with approximately log n comparisons. This is designated using big O notation, as O(log n).


How About Sorting Objects?

The same algorithm applies to arrays containing objects. If you have a sorted array of SoccerPlayer objects (sorted based on a specific attribute) and you need to find a specific player, then you would access the SoccerPlayer object's attribute, and compare with the target value.
